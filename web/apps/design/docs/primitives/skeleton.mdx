Use `Skeleton` when async data will fill a known layout, such as a table, form, or card grid. It reserves space for the final content so the page doesn't jump when data arrives.

## Usage

```tsx
import { Skeleton } from "@unkey/ui";

<Skeleton className="h-4 w-32" />;
```

## Best practices

Skeleton is one of three loading primitives in `@unkey/ui`. Use it when async data will fill a known layout. Don't use skeletons on permanent UI or empty states, because that suggests content is still coming.

For a single action, use `<Loading type="spinner" />`. For an indeterminate wait, use `<Loading type="dots" />`.

### Sizing

Size the skeleton to match the content it replaces. `className="h-4 w-32"` creates a 128px text bar and prevents layout shift when the real text loads.

Use `rounded-full` for circular avatars. The default `rounded-sm` already matches text bars, buttons, and badges in the dashboard.

### Accessibility

Set `aria-busy="true"` on loading regions, not skeletons. Announce completion with `aria-live="polite"` on the destination container. Don't put focusable controls inside a skeleton.

Skeletons respect `prefers-reduced-motion` through `motion-reduce:animate-none`.
